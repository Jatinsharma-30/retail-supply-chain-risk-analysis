# SQL Analysis

## Objective

The SQL component of this project focuses on transforming raw inventory data into actionable business insights through structured querying and analytical techniques.

## Database

Database Name: retail_inventory

Table Name: retail_store_inventory

Total Records: 73,100

## Analysis Areas

### 1. Sales Analysis

Business Questions:

- Which categories generate the highest sales?
- Which regions contribute most to sales volume?
- How does seasonality affect sales performance?

Queries Included:

- Total Sales
- Sales by Category
- Sales by Region
- Seasonal Performance

---

### 2. Inventory Analysis

Business Questions:

- Which products face stock shortage risk?
- Which categories hold the highest inventory value?
- How efficiently is inventory managed?

Queries Included:

- Inventory Value Analysis
- High Risk Inventory Analysis
- Inventory Coverage Analysis

---

### 3. Demand Forecast Analysis

Business Questions:

- How accurate are demand forecasts?
- Which categories experience the highest forecasting error?
- Which regions require forecasting improvements?

Queries Included:

- Forecast Error Analysis
- Category-wise Forecast Accuracy
- Regional Forecast Performance

---

### 4. Pricing Analysis

Business Questions:

- How do product prices compare with competitor prices?
- What is the impact of discounts on sales?
- Which categories show the largest pricing gaps?

Queries Included:

- Competitor Price Comparison
- Discount Impact Analysis
- Price Difference Analysis

---

### Advanced SQL Concepts

The project demonstrates the use of:

- Common Table Expressions (CTEs)
- Window Functions
- RANK()
- DENSE_RANK()
- CASE Statements
- Aggregations
- Grouping Operations

## Key Outcomes

- Identified inventory risk patterns
- Evaluated demand forecasting performance
- Measured pricing competitiveness
- Analyzed seasonal demand trends
- Generated insights for supply chain optimization
